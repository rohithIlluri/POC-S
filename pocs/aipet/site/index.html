WHAT IT IS HOW IT WORKS STARTERS THE DEX INSTALL PRIVACY

What it is

A creature lives in your terminal. It feeds on the tokens you already burn, and it thrives on quality, not volume — good engineering hygiene raises a healthy, rare creature. Wasteful habits make it sluggish. It's a game you play by doing your actual job well.

🐣 Raise a CodelingAn egg hatches after 3 active days into one of three starter lines — picked from how you actually worked. It levels, grows five stats, and evolves at 12 and 30.
🪙 Get coached, get savingsThe same engine flags Opus overuse, low cache reuse, and context bloat — with estimated dollar savings. Efficiency advice is your pet's diet.
🏆 Keep scoreA local leaderboard tracks your top projects, best cache-reuse days, streaks, and personal records. Yours alone — computed on-device.
🔒 100% local, $0 foreverIt reads the session logs your tools already write to disk. No servers, no accounts, no telemetry — and it never calls a model, so playing costs zero tokens.

How it works

Claude Code and Codex already write session logs to your disk. Codelings folds them into a daily "digest," classifies the day's diet, and advances a deterministic simulation — one tick per calendar day.

 you work with Claude Code / Codex
              │
              ▼  reads logs already on your disk — 0 tokens, 0 network
   usage events → daily digest
              │
              ▼  diet verdict: balanced · junk food · rich food · overeating
 XP · health · mood · stat growth · evolution checks
              │
              ▼
  your terminal: the pet reacts, the journal explains why

Tokens are food — the diet

LOW CACHE REUSE
Junk food. Re-sent uncached prompts digest poorly — health dips, pet looks tired.
BIG-MODEL OVERUSE
Rich food. Fine sometimes; a steady diet makes it lethargic.
HUGE CONTEXTS
Overeating. "Token bloat" status until habits improve.
OVER BUDGET
Foraging limit. Past your soft daily budget, extra tokens give zero XP.
HEALTHY DAY
Balanced diet. Full XP, mood up, evolution progress.

Kind by design: vacations never hurt. After 7 quiet days your pet hibernates — and wakes up happy, zero guilt, whenever you're back.

Three starter lines

Your first three active days decide the egg. Deep focused sessions, fast cache-heavy iteration, or breadth across projects and models — the egg is watching.

Encounters are catch-by-doing: a wild Codeling appears on real events — a new project, a new model, an unusually clean day — and joins you if you finish the day well. The odds only ever improve with better habits, never with more spend.

Install & play

OPTION A — GO INSTALL (RECOMMENDED)
go install github.com/rohithIlluri/POC-S/pocs/aipet/cmd/aipet@latest
OPTION B — PREBUILT BINARY
# darwin / linux / windows × amd64 / arm64, with SHA-256 checksums
github.com/rohithIlluri/POC-S/releases/latest
FIRST RUN
aipet status      # one-shot: collects your existing logs + summary
aipet             # the pet (TUI): Pet · Overview · Suggestions · Records
aipet daemon      # background watcher — keeps the pet growing
aipet leaderboard # rankings + personal records (--json for scripts)
OPTIONAL CONFIG
aipet config daily_budget_usd 15      # soft budget — the pet's "foraging limit"
aipet config collect_interval_min 5   # daemon re-scan cadence

An egg starts warming on your first run. Three active days later, it hatches. What it becomes is up to how you work.

Private by construction

Codelings reads the session logs Claude Code and Codex already write to your disk (~/.claude/projects, ~/.codex/sessions). No proxy, no interception — your code and prompts never leave the machine. All state lives in ~/.aipet. Session-log content is treated as untrusted input and sanitized before display.
